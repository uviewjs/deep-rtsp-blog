RTSP配置规则

大华（Dahua）

RTSP格式

rtsp://username:password@ip:port/cam/realmonitor?channel=1&subtype=0

参数说明

  • username：设备用户名（如 admin）。
  • password：设备密码。
  • ip：设备IP地址（如 192.168.1.100）。
  • port：端口号，默认 554，可省略。
  • channel：通道号，从 1 开始（如 channel=2 表示第2通道）。
  • subtype：码流类型，0 为主码流，1 为辅码流。

示例

  • 主码流（通道1）：
    rtsp rtsp://admin:admin@127.0.0.1/cam/realmonitor?channel=1&subtype=0

海康威视（Hikvision）

1. 旧版规则（2012年前设备）

rtsp://<username>:<password>@<ipaddress>:<port>/<videotype>/ch<id>/<streamtype>/av_stream
  • videotype：视频编码格式（如 h264、mpeg4）。
  • ch：通道号（如 ch1 表示通道1）。
  • streamtype：码流类型，main 为主码流，sub 为辅码流。

示例： 

rtsp://admin:admin@127.0.0.1/h264/ch1/main/av_stream  # 主码流
rtsp://admin:admin@127.0.0.1/mpeg4/ch2/sub/av_stream   # 辅码流

2. 新版规则（预览取流）

rtsp://<username>:<password>@<ipaddress>:<port>/Streaming/Channels/<id>(?parm1=value1&parm2=value2)
  • id：通道号+0+码流类型，码流类型 1 为主码流，2 为辅码流（如 101 表示通道1主码流，102 表示通道1辅码流）。
  • parms：可选参数，如 transportmode=unicast（单播，默认）或 multicast（多播）。

示例： 

rtsp://admin:admin@127.0.0.1/Streaming/Channels/101  # 通道1主码流

3. 回放取流

rtsp://<username>:<password>@<ipaddress>:<port>/Streaming/tracks/<id>(?starttime=...&endtime=...)
  • 时间格式：YYYYMMDD"T"HHmmSS.fractionZ（如 20231001T123000Z）。

示例： 

rtsp://admin:admin@127.0.0.1/Streaming/tracks/101?starttime=20231001t123000z&endtime=20231001t130000z

宇视（Uniview）

摄像头取流

rtsp://用户名:密码@ip:port/video{num}  # num=1（主码流），2（辅码流），3（第三码流）

示例： 

rtsp://admin:admin@192.168.8.8/video1  # 主码流

NVR取流

rtsp://用户名:密码@ip:port/unicast/c<channel>/s<stream_type>/live
  • channel：通道号（如 c1 表示通道1）。
  • stream_type：0 为主码流，1 为辅码流。

示例： 

rtsp://admin:admin@192.168.8.7/unicast/c1/s0/live  # 通道1主码流

华为（Huawei）

摄像头取流

rtsp://用户名:密码@ip:port/LiveMedia/ch<id>/Media{type}
  • ch：通道号（如 ch1 表示通道1）。
  • Media{type}：Media1 为主码流，Media2 为辅码流。

示例： 

rtsp://admin:admin@192.168.2.98/LiveMedia/ch1/Media1  # 主码流

NVR取流

rtsp://username:password@ip:port/rtsp/streaming?channel=A&subtype=B
  • A：通道号（如 1 表示通道1）。
  • B：码流类型，0 为主码流，1 为辅码流。

示例： 

rtsp://admin:admin@192.168.2.98/rtsp/streaming?channel=1&subtype=0

天地伟业（Tiandy）

RTSP格式

rtsp://用户名:密码@ip:port/{通道号}/{码流类型}
  • 通道号：直接填写数字（如 1 表示通道1）。
  • 码流类型：1 为主码流，2 为辅码流。

示例： 

rtsp://admin:admin@192.168.2.98/1/1  # 通道1主码流

通用验证工具：VLC媒体播放器

  1. 打开VLC，选择 媒体 > 打开网络串流。
  2. 输入RTSP地址（如 rtsp://admin:admin@192.168.1.100/cam/realmonitor?channel=1&subtype=0）。
  3. 点击 播放 验证是否正常拉流。

注意事项

  1. 端口问题：部分厂商端口非默认 554（如华为NVR可能使用HTTPS端口），需确认设备配置。
  2. 权限问题：确保用户名密码正确，且设备启用RTSP服务（部分厂商需在后台手动开启）。
  3. Onvif协议：部分厂商支持Onvif协议（如华为、天地伟业），可通过Onvif地址（如 http://ip/onvif/device_service）发现设备后获取RTSP地址。
